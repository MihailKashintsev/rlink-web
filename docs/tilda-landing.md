# Веб-оболочка Rlink на Tilda (лендинг + ссылки на каналы)

**Готовые материалы:**

- **[tilda-copy-ready.md](tilda-copy-ready.md)** — тексты для блоков Tilda (герой, фичи, FAQ, подвал, страница канала).  
- **[tilda-privacy-policy-ru.md](tilda-privacy-policy-ru.md)** — черновик политики конфиденциальности (подставьте контакты и URL, проверьте с юристом).

Полноценный мессенджер в браузере здесь **не** делаем: на Tilda размещается **сайт-витрина** (установка, описание, политика, контакты) и **точка входа по ссылкам** вида `https://…/rlink/channel/<id>`, которые уже зашиты в приложении (`RlinkDeepLink.installWebBase` в `lib/utils/rlink_deep_link.dart`).

---

## 1. Что поменять в коде при смене домена

Сейчас база веб-ссылок:

- `https://rendergames.online/rlink`

Если на Tilda будет другой домен или путь — обновите константу **`RlinkDeepLink.installWebBase`** и пересоберите приложения. На Android проверьте **`AndroidManifest.xml`**: intent-filter для `https` с `host` и `pathPrefix` должны совпадать с реальным URL.

## 2. Карта сайта (рекомендуемая)

| URL (пример) | Назначение |
|--------------|------------|
| `/rlink` или `/rlink/` | Главная: что такое Rlink, кнопки в магазины, скриншоты |
| `/rlink/channel` | Не обязательна отдельно — см. п. 4 |
| `/rlink/privacy` | Политика конфиденциальности (часто нужна для стора) |
| `/rlink/terms` | Условия использования (по желанию) |

Страница **каждого** канала по пути `/rlink/channel/<id>` на чистой Tilda без кода обычно **не генерируется автоматически**. Варианты:

- **Zero Block** на одной странице с URL вида `/rlink/channel` + скрипт, который читает хвост пути (ниже).
- Либо **отдельный лёгкий хостинг** (Cloudflare Workers, GitHub Pages) только для префикса `/rlink/channel/*`, а остальное — Tilda.

## 3. Главная страница — тексты для блоков (можно копировать в Tilda)

**Шапка**

- Заголовок: **Rlink**
- Подзаголовок: **Офлайн-мессенджер через Bluetooth и интернет-реле. Каналы, группы, личные чаты.**

**Блок «Скачать»**

- Кнопка: **Скачать для Android** → ссылка на **APK на вашем сайте** (в Google Play приложения пока нет).
- Кнопка: **iPhone / iPad** → сборка с сайта (App Store нет).
- Кнопка: **Для Windows / macOS** → свои релизы или GitHub Releases.

**Блок «Возможности» (коротко)**

- Обмен сообщениями поблизости по **Bluetooth**.
- **Реле** в интернете, когда пиров рядом нет.
- **Каналы** с постами и подписками.
- **Группы** и личные чаты с шифрованием там, где это реализовано в приложении.

**Блок «Каналы»**

- Текст: **Пригласительная ссылка открывает канал в приложении, если вы уже подписаны или получили приглашение. Иначе сначала установите Rlink и добавьте канал через поиск или приглашение от администратора.**

**Подвал**

- Контакты, ссылки на политику, копирайт.

## 4. Страница открытия канала по ссылке (важно для шаринга)

Приложение шарит текст вида:

`Канал «…» в Rlink` + `https://<ваш-домен>/rlink/channel/<channelId>`

На этой странице пользователю нужно:

1. Понять, что открыть нужно **приложение Rlink**.
2. Получить кнопки **Установить** / **Открыть** (по платформе).

### Вариант: Zero Block (HTML + JS)

Создайте в Tilda страницу с постоянным адресом, совпадающим с тем, как приложение формирует ссылку. Если Tilda не даёт вложенный путь `rlink/channel/:id` для всех id, используйте **одну** страницу с URL `rlink/channel` и в настройках сайта/страницы включите поддержку «лишнего» хвоста пути **или** перенесите обработку на субдомен/воркер.

Минимальная логика скрипта:

- Из `window.location.pathname` взять сегмент после `channel` — это `channelId`.
- Показать заголовок: «Канал в Rlink» и мелким шрифтом id (или без id).
- Кнопки: магазины приложений.
- Ссылка **Открыть в приложении**: `rlink://channel/<channelId>` (для тех ОС, где кастомная схема открывается из браузера).

Пример скелета (проверьте путь под вашу разметку URL):

```html
<div id="rlink-channel-landing">
  <h1>Открыть канал в Rlink</h1>
  <p id="ch-hint">Установите приложение и откройте ссылку снова — или используйте кнопку ниже.</p>
  <p><a id="open-app" href="#">Открыть в приложении</a></p>
  <p><a href="https://ВАШ-САЙТ/rlink">Android — загрузка (APK)</a></p>
  <p><a href="https://ВАШ-САЙТ/rlink">Установка (сайт)</a></p>
</div>
<script>
(function () {
  var parts = location.pathname.split('/').filter(Boolean);
  var i = parts.indexOf('channel');
  var id = (i >= 0 && parts[i + 1]) ? parts[i + 1] : '';
  var a = document.getElementById('open-app');
  if (id && a) {
    a.href = 'rlink://channel/' + encodeURIComponent(id);
  }
})();
</script>
```

Подставьте ссылку на страницу загрузки (магазинов нет).

## 5. App Links (Android) и Universal Links (iOS)

Чтобы по тапу по **https**-ссылке сразу открывалось приложение:

- **Android**: файл **Digital Asset Links** на домене + уже настроенный `intent-filter` в приложении.
- **iOS**: файл **apple-app-site-association** + включённые Associated Domains в Xcode.

Tilda сама эти файлы не генерирует: их кладут в корень домена или в `.well-known/` (как требует Apple/Google). Обычно это делает **DNS/хостинг** или **Cloudflare**, а Tilda висит на поддомене или на том же домене с проксированием — уточните у поддержки Tilda, куда залить статические JSON без редиректа.

## 7. Связь с репозиторием

| В коде | Назначение |
|--------|------------|
| `lib/utils/rlink_deep_link.dart` | База URL, шаринг текста, разбор ссылок |
| `lib/services/rlink_deep_link_service.dart` | Открытие канала из `rlink://` и https в приложении |
| `android/.../AndroidManifest.xml` | intent-filter для схемы и https |

После переноса сайта на Tilda достаточно обновить константу и при необходимости нативные ассоциации домена — логика приложения меняться не должна.
